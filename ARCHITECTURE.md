## ğŸ¯ Design Decisions

### Why Separate Namespaces?

- **Isolation**: Each component has its own namespace for resource isolation
- **Security**: Network policies can restrict cross-namespace access
- **Organization**: Clear separation of concerns
- **RBAC**: Fine-grained access control per namespace

### Why These Replacements?

| Decision | Rationale |
|----------|-----------|
| **Redpanda over Kafka** | No ZooKeeper dependency, simpler ops, Kafka compatible |
| **Dragonfly over Redis** | Better performance, multi-threaded, Redis compatible |
| **SeaweedFS over MinIO** | Simpler architecture, better for small files, S3 compatible |
| **MySQL (not PostgreSQL)** | OpenIM official support, better tested, wider adoption |

### Why GKE?

- **Managed Control Plane**: Google manages Kubernetes masters
- **Auto-scaling**: Both pod and cluster level
- **Integration**: Native GCP integration (Load Balancer, Storage, IAM)
- **Reliability**: Google SLA and uptime guarantees
- **Security**: Workload Identity, Binary Authorization, etc.
